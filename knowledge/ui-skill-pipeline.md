# UI/UX Skill Pipeline

Canonical reference for the multi-skill UI quality pipeline used by the executor and frontend-designer agents. Defines which skills run, in what order, and how they differ based on the branding choice.

---

## 1. Overview

Every generated frontend passes through a multi-skill quality pipeline that produces premium, polished UI. The pipeline has two variants:

- **Lean pipeline** -- for projects using Integra official branding (design tokens are pre-defined)
- **Full pipeline** -- for projects using custom branding (design system must be generated first)

The branding choice is stored in `.integra/config.json` under the `branding` field (`"integra"` or `"custom"`).

---

## 2. Skills in the Pipeline

| # | Skill | What It Does | Required For |
|---|-------|-------------|-------------|
| 1 | `web-design-guidelines` | Layout fundamentals, typography scale, responsive patterns, whitespace rules | Both pipelines |
| 2 | `ui-ux-pro-max` | Generates a complete custom design system -- 50+ visual styles, 161 color palettes, font pairings | Custom pipeline only |
| 3 | `taste-skill` | Premium visual feel -- spacing rhythm, visual hierarchy, content density, polish | Both pipelines |
| 4 | `shadcn-ui` | Consistent component usage with the shadcn/ui library -- variant patterns, composition | Both pipelines |
| 5 | `ui-animation` | Smooth transitions (150-300ms), hover effects (`translateY(-2px)`), list/page animations | Both pipelines |
| 6 | `frontend-patterns` | Component architecture -- atomic design, composition patterns, state management | Both pipelines |
| 7 | `react-dev` | React 18-19 patterns -- Server Components, Suspense boundaries, hooks best practices | Both pipelines |

---

## 3. Pipeline Order

### Lean Pipeline (Integra Branding)

```
web-design-guidelines    Establish layout + typography rules
        |
  frontend-patterns      Define component architecture
        |
    react-dev            Apply React best practices
        |
    shadcn-ui            Build components with shadcn/ui
        |
  ui-animation           Add transitions and motion
        |
   taste-skill           Final polish pass (spacing, hierarchy)
```

**Skips:** `ui-ux-pro-max` (design tokens already defined in `knowledge/design-systems/integra-brand.md`)

**Design tokens source:** `knowledge/design-systems/integra-brand.md`

### Full Pipeline (Custom Branding)

```
web-design-guidelines    Establish layout + typography rules
        |
  ui-ux-pro-max          Generate custom design system
        |                 (writes to knowledge/design-systems/custom-brand.md)
  frontend-patterns      Define component architecture
        |
    react-dev            Apply React best practices
        |
   taste-skill           Apply premium visual feel to custom palette
        |
    shadcn-ui            Build components with custom theme tokens
        |
  ui-animation           Add transitions and motion
```

**Design tokens source:** `knowledge/design-systems/custom-brand.md` (generated by ui-ux-pro-max)

## 4. When Skills Are Applied

| Build Phase | Skills Applied |
|-------------|---------------|
| Phase 2: Frontend Scaffold | `web-design-guidelines`, `frontend-patterns`, `react-dev` |
| Phase 3: Contract Integration | `react-dev` (hooks patterns) |
| Phase 4: UI Polish | `taste-skill`, `shadcn-ui`, `ui-animation`, `ui-ux-pro-max` (custom only) |
| Phase 6: Polish & Verify | `taste-skill` (final pass), `ui-animation` (final pass) |

---

## 5. Stitch Integration

When Google Stitch AI is used for UI generation (see `knowledge/stitch-integration.md`), the pipeline adjusts:

1. **Stitch generates the visual layout** -- screen designs with HTML export
2. **The skill pipeline applies to the React rebuild** -- executor uses Stitch HTML as visual spec, then applies all pipeline skills during the React component implementation
3. **Stitch does not replace skills** -- it provides layout inspiration; the skills ensure code quality, accessibility, animation, and polish

The pipeline order remains the same regardless of whether Stitch is used.

---

## 6. Quality Checklist

After the pipeline runs, every frontend component should pass:

- [ ] Layout follows `web-design-guidelines` spacing and grid rules
- [ ] Component architecture follows `frontend-patterns` composition rules
- [ ] React patterns follow `react-dev` Server Component and hooks best practices
- [ ] All components use `shadcn-ui` as base (no custom components from scratch)
- [ ] Animations are subtle and purposeful per `ui-animation` (150-300ms, no flashy effects)
- [ ] Visual hierarchy passes `taste-skill` review (spacing rhythm, content density, polish)
- [ ] Design tokens match the selected branding source (integra-brand.md or custom-brand.md)
- [ ] Dark theme is default with proper surface layering
- [ ] Responsive at 375px (mobile), 640px (tablet), 1024px+ (desktop)
